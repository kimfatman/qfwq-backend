# 企服外勤代办宝 · 后端架构设计

## 一、技术选型

| 层面 | 选型 | 说明 |
|------|------|------|
| 运行时 | Node.js 20 LTS | 轻量高效，适合I/O密集型 |
| Web框架 | Express 4.x | 生态成熟，中间件丰富 |
| 数据库 | MySQL 8.0 | 腾讯云CDB支持，关系型存储 |
| ORM | Prisma | 类型安全，迁移方便 |
| 认证 | JWT (access + refresh) | 与前端 token 机制对齐 |
| 文件存储 | 腾讯云 COS | 对象存储，CDN加速 |
| 缓存 | Redis 7.x | 会话/验证码/限流/热点数据 |
| 部署 | Docker + Nginx | 腾讯云轻量/CVM |
| 进程管理 | PM2 | 守护进程 + 日志管理 |

## 三、数据库设计（MySQL）

### 3.1 表关系总览

```
users ──┬── 1:N ──> orders
        ├── 1:N ──> addresses
        ├── 1:N ──> reviews
        └── 1:N ──> feedbacks

runners ──┬── 1:N ──> quotes
          ├── 1:N ──> orders (assigned_runner_id)
          └── 1:N ──> reviews

services ──┬── 1:N ──> orders
           └── 1:1 ──> service_pricing

orders ──┬── 1:N ──> quotes
         ├── 1:N ──> order_progress
         ├── 1:N ──> order_materials
         └── 1:N ──> reviews

banners (独立)
agreements (独立)
system_config (独立)
```

## 五、核心业务逻辑

### 5.1 微信登录流程
```
小程序 wx.login() → code
      ↓
后端 POST /auth/wx-login {code}
      ↓
后端调用微信 code2Session API → openid + session_key
      ↓
查找/创建用户 → 生成 JWT (access 2h + refresh 7d)
      ↓
返回 {token, refreshToken, userInfo}
```

### 5.2 发单 → 报价 → 选人 流程
```
企业创建订单 → status=pending → 推送给匹配外勤
      ↓
外勤查看并报价 → quotes 表插入 → status=quoted
      ↓
企业查看报价列表 → 选择外勤
      ↓
POST /quote/select/:id → order.status=assigned → 通知外勤
      ↓
外勤开始办理 → order.status=in_progress → 更新进度
      ↓
外勤完成 → 通知企业确认
      ↓
企业确认完成 → order.status=completed → 引导评价
```

### 5.3 虚拟号方案
- 使用腾讯云「号码 privacy protection」服务
- 绑定：外勤接单后绑定 AXB 关系
- 解绑：订单完成/取消后自动解绑
- 降级：API失败时直接返回外勤真实号码

### 5.4 文件上传
- 前端通过 `uni.uploadFile` 上传
- 后端接收后转存至腾讯云 COS
- 返回 COS 文件 URL
- 支持格式：jpg/png/pdf，单文件 ≤ 10MB
- COS 路径规范：`orders/{orderId}/{timestamp}_{random}.{ext}`

### 7.3 Nginx 配置要点
```nginx
server {
    listen 443 ssl http2;
    server_name api.yourdomain.com;

    ssl_certificate     /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        client_max_body_size 20m;  # 文件上传
    }

    # 上传文件代理
    location /uploads/ {
        proxy_pass http://127.0.0.1:3000;
    }
}
```
